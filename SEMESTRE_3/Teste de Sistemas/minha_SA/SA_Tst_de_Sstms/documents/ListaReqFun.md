
# Listando os Requisitos Funcionais da API:

| #Nº | RF - Req. Func. | - | Descrição |
|:---:| :--------------- |:-:|:----------|
| #01 | RF - Cadastrar moradores | - | O sistema deve permitir o cadastro de um morador informando o nome, bloco e número do apartamento | 
| #02 | RF - Listar moradores | - | O sistema deve permitir a listagem de todos os moradores cadastrados, ordenados por ID | 
| #03 | RF - Buscar morador por ID | - | O sistema deve permitir consultar um morador específico a partir do seu ID | 
| #04 | RF - Atualizar dados do morador | - | O sistema deve permitir atualizar os dados de um morador existente |
| #05 | RF - Deletar morador | - | O sistema deve permitir remover um morador pelo ID |
| #06 | RF - Criar recado | - | O sistema deve permitir criar um recado informando |
| #07 | RF - Listar recados | - | O sistema deve permitir listar todos os recados cadastrados |
| #08 | RF - Buscar recado por ID | - | O sistema deve permitir consultar um recado específico pelo ID |
| #09 | RF - Atualizar recado | - | O sistema deve permitir atualizar um recado existente |
| #10 | RF - Deletar recado | - | O sistema deve permitir excluir um recado pelo ID |
| #11 | RF - Associar recado a um morador. | - | O sistema deve garantir que todo recado esteja vinculado a um morador existente |
