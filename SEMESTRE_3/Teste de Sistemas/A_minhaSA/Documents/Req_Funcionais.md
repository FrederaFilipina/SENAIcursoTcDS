# Listando os Requisitos Funcionais da API: Mural Condômino

| #Nº | Req. Func. | - | Descrição |
|-----|------------|---|-----------|
| 1 | Login e autenticação de moradores | - | O sistema deve permitir que moradores acessem a aplicação por meio de usuário e senha, validando credenciais únicas. |
| 2 | Cadastro e manutenção de moradores | - | O sistema deve possibilitar incluir, atualizar, consultar e excluir dados dos moradores (nome, bloco, apartamento, usuário e senha). |
| 3 | Cadastro e manutenção de recados | - | O sistema deve permitir que moradores registrem recados, atualizem informações, consultem e excluam quando necessário. |
| 4 | Classificação de recados por tipo | - | Cada recado deve ser categorizado obrigatoriamente como Aviso, Comunicado ou Encomenda. |
| 5 | Controle de status dos recados | - | O sistema deve permitir alterar o status dos recados entre ativo e inativo, garantindo o gerenciamento adequado. |
| 6 | Registro automático da data/hora de criação | - | O sistema deve registrar automaticamente a data e hora de criação de cada recado. |
| 7 | Exclusão em cascata de recados | - | Ao excluir um morador, todos os recados vinculados a ele devem ser automaticamente removidos. |
| 8 | Consulta de recados | - | O sistema deve permitir consultar recados filtrando por morador, tipo e status. |
